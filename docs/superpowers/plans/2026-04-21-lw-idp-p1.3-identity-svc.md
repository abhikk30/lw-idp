# lw-idp P1.3 ‚Äî identity-svc end-to-end Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Each subagent prompt carries the tactical file content.

**Goal:** Turn the identity-svc skeleton into a real authenticating service. Demo: POST a valid Dex-issued OIDC id_token to identity-svc ‚Üí user row upserted in Postgres ‚Üí `idp.identity.user.created` event on NATS. Browser flow (`/auth/callback`) works end-to-end against Dex, issuing a session cookie.

**Architecture:**
- Identity-svc writes to its own Postgres database (`identity`) on the shared CNPG cluster via a CNPG `Database` CR.
- gRPC surface via **ConnectRPC** (Connect protocol over HTTP/1.1) plugged into Fastify on the same port as the HTTP surface (4001). Simpler than native gRPC h2 and curl-testable.
- Transactional outbox: every mutating handler writes domain row + `outbox` row in one Postgres tx; a background publisher polls for unpublished rows, publishes to NATS JetStream, marks as published.
- OIDC verification uses `jose`'s `createRemoteJWKSet` pointed at Dex's `/keys` endpoint.
- Temporary `/auth/login` + `/auth/callback` HTTP routes on identity-svc so we can demo the flow end-to-end. These move to `gateway-svc` in Plan 1.5.

**Tech stack:** Fastify 5, `@connectrpc/connect`, `@connectrpc/connect-fastify`, `@bufbuild/protobuf`, Drizzle ORM, postgres.js, `jose` (JWKS), NATS JetStream (via `nats` package from P1.1 `@lw-idp/events`), Zod, `@lw-idp/service-kit` (from P1.2, extended).

## Conventions

- Follow existing patterns from P1.1/P1.2: strict TS, `rootDir:"."`, Biome, Conventional Commits.
- Every mutating handler: domain write + outbox insert in one Drizzle transaction (`db.transaction(async (tx) => { ... })`).
- Every gRPC response conforms to the generated Proto types ‚Äî no manual JSON shaping.
- Every new env var read via `zod` schema in `src/config.ts` at boot.
- Test containers: reuse `@lw-idp/testing`'s `startPostgres()` + `startNats()`.
- JWKS mock: for unit tests use `jose`'s own `exportJWK` + a static key set; for integration tests run a tiny node http server that serves `/keys` and `/token` like Dex.

## Deliverables when complete

1. `grpcurl -plaintext identity-svc.lw-idp:80 lwidp.identity.v1.IdentityService/ListUsers` (or equivalent ConnectRPC-via-curl) returns a list.
2. Hit `identity-svc`'s `/auth/login` ‚Üí redirects to Dex ‚Üí after GitHub OAuth returns to `/auth/callback?code=...` ‚Üí Dex issues id_token ‚Üí identity-svc verifies, upserts user, publishes `idp.identity.user.created` on NATS ‚Üí returns 200 with a session cookie.
3. `nats sub 'idp.identity.>'` in another shell shows the event within ~100ms of the callback hit.
4. CI green (all P1.2 jobs + no regressions).
5. Tag `p1.3-identity-svc` on the final commit.
