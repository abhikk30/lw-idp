# lw-idp P1.5 — gateway-svc (BFF) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development. Tactical task content carried inline per subagent prompt.

**Goal:** `gateway-svc` becomes the real browser entry point. Browser flow is reachable via ingress at `portal.lw-idp.local`: hit `/auth/login` → Dex → callback → **session cookie backed by Dragonfly** → subsequent REST calls aggregate across identity / catalog / cluster ConnectRPC services. Rate-limited, idempotency-key aware, OpenAPI-contract-first.

**Architecture:**
- Fastify 5 + `@lw-idp/service-kit` + OpenAPI-first spec under `packages/contracts/openapi/gateway.yaml`.
- Session store: Dragonfly via `ioredis` (HA-safe — multi-replica).
- OIDC: reuse `@lw-idp/auth`'s `createOidcVerifier`; move `/auth/login` + `/auth/callback` PKCE state to Dragonfly (retires P1.3 I3 — identity-svc scales back to 2 replicas).
- Upstream clients: ConnectRPC node transport over HTTP/1.1 to each of identity-svc / catalog-svc / cluster-svc.
- Rate limit: `@fastify/rate-limit` (Redis-backed, keyed by session+route).
- Idempotency: `Idempotency-Key` header on unsafe writes → Dragonfly hash, 24h TTL, replays return stored response.
- Ingress: `ingress-nginx` routes `portal.lw-idp.local` host to the gateway Service; TLS via cert-manager.

**Spec refs:** `docs/superpowers/specs/2026-04-20-lw-idp-foundation-design.md` §6.2, §7.1, §7.2, §7.6.
**Predecessor:** P1.4 (tag `p1.4-catalog-cluster`).

## Deliverables

1. `curl -L http://portal.lw-idp.local/auth/login` bounces through Dex and returns with a set-cookie.
2. With that cookie: `curl -H 'Cookie: lw-sid=…' http://portal.lw-idp.local/api/v1/me` → `{ user, teams }`.
3. `curl -X POST -H 'Idempotency-Key: foo' …/api/v1/services` creates once, replays return the cached response.
4. Rate limit returns 429 after N requests in window.
5. identity-svc runs at `replicaCount: 2` without state-store split-brain.
6. CI green, tag `p1.5-gateway-svc`.
